# 6. Information Architecture

---

## ⑥ Information Architecture

MVP 원칙에 맞춰 화면 구조를 **최소한으로** 설계합니다. 탭 개수, 화면 depth 모두 의도적으로 줄였습니다 — 화면을 많이 만들수록 "화면을 안 봐도 되는 앱"이라는 정체성과 모순되기 때문입니다.

---

### 1. 전체 앱 구조 (Sitemap)

```
Haptic Football (iOS App)
│
├── [최초 실행 전용] 온보딩 플로우
│   ├── 스플래시 / 컨셉 소개
│   ├── 햅틱 프리뷰 체험
│   ├── 리그 선택
│   ├── 관심 팀 선택
│   └── 알림 권한 요청
│
├── [메인] 탭 구조 (온보딩 완료 후 진입)
│   ├── Tab 1. 홈 (경기 목록)
│   ├── Tab 2. 팔로우 중인 경기
│   └── Tab 3. 설정
│
└── [모달/상세] 탭에 속하지 않는 화면
    ├── 경기 상세 화면
    ├── 경기 결과 화면 (종료 후)
    └── 햅틱 패턴 가이드 화면
```

---

### 2. 탭 구조 상세

MVP는 **3개 탭**으로 제한합니다. (스포츠 앱들이 흔히 가진 "뉴스", "커뮤니티", "랭킹" 탭 등은 ④ Scope에서 이미 배제)

```
┌─────────────────────────────────────┐
│                                       │
│         [현재 화면 콘텐츠]              │
│                                       │
├─────────────────────────────────────┤
│   🏠 홈    ⚡ 팔로잉    ⚙️ 설정         │
└─────────────────────────────────────┘
```

| 탭 | 역할 | MVP 필요성 |
| --- | --- | --- |
| 홈 | 오늘/예정 경기 목록, 관심 팀 경기 하이라이트 | 신규 경기 발견 및 등록의 진입점 |
| 팔로잉 | 현재 팔로우 중인 경기 리스트 + 상태(예정/진행중/종료) | "지금 뭘 추적 중인지" 한눈에 확인 |
| 설정 | 관심 팀 관리, 알림 on/off, 햅틱 패턴 가이드 진입 | 최소한의 개인화 관리 |

---

### 3. 화면별 페이지 구조 (Page Structure)

#### 3-1. 온보딩 플로우 (최초 1회만)

```
[스플래시]
  "화면을 보지 않아도, 경기의 흐름을 몸으로 느껴보세요"
  → CTA: "시작하기"
       ↓
[햅틱 프리뷰]
  - 골 / VAR / 카드 진동 샘플 3개
  - 각 버튼 탭 → 실제 진동 재생
  → CTA: "다음"
       ↓
[리그 선택]
  - EPL / FIFA World Cup 카드 2개
  - 단일 선택
  → CTA: "다음"
       ↓
[관심 팀 선택]
  - 선택 리그 내 팀 그리드
  - 1개 이상 선택
  → CTA: "다음"
       ↓
[알림 권한 요청]
  - iOS 네이티브 권한 팝업
  → 허용/거부 후 자동 진행
       ↓
[메인 탭 진입] (홈 탭 기본 선택)
```

#### 3-2. 홈 탭

```
팔로잉
├── 진행중인 경기 (있을 경우 최상단, 실시간 상태 뱃지)
├── 예정된 경기 (팔로우했지만 아직 시작 안 함)
├── 최근 종료된 경기 (최근 3~5개 정도만, 스코어 요약)
└── 각 항목 탭 → 경기 상세 화면(모달)

* 비어있을 때: "아직 팔로우 중인 경기가 없어요 → 홈에서 경기를 찾아보세요" (홈 탭으로 유도)
```

#### 3-3. 팔로잉 탭

```
팔로잉
├── 진행중인 경기 (있을 경우 최상단, 실시간 상태 뱃지)
├── 예정된 경기 (팔로우했지만 아직 시작 안 함)
├── 최근 종료된 경기 (최근 3~5개 정도만, 스코어 요약)
└── 각 항목 탭 → 경기 상세 화면(모달)

* 비어있을 때: "아직 팔로우 중인 경기가 없어요 → 홈에서 경기를 찾아보세요" (홈 탭으로 유도)
```

#### 3-4. 경기 상세 화면 (모달, 탭에 속하지 않음)

```
경기 상세
├── 상단: 팀 로고, 스코어(진행중/종료 시), 경기 상태(예정/진행중/종료)
├── 팔로우 토글 버튼 ("이 경기 팔로우하기" / "팔로우 중")
├── (진행중/종료 시) 주요 이벤트 텍스트 요약
│     - 몇 분에 골/카드/VAR 발생했는지 간단 리스트
│     - ※ 스크롤 타임라인이 아니라 "짧은 리스트" 수준으로 최소화
└── 하단: 닫기
```

> 여기서 "주요 이벤트 텍스트 요약"이 들어간 이유: 진동으로 흐름은 느꼈지만, 이후 "무슨 일이 있었는지 정리해서 보고 싶다"는 니즈는 자연스러운 확인 행동입니다. 이건 실시간으로 계속 보는 스코어보드가 아니라, **사용자가 능동적으로 앱을 열었을 때만** 보이는 결과 요약이라 Screenless 철학과 충돌하지 않습니다.
> 

#### 3-5. 경기 결과 화면 (경기 종료 후, 상세 화면과 통합 가능)

```
* 별도 화면이라기보다 "경기 상세" 화면이 종료 상태일 때의 뷰로 처리
* 최종 스코어 + 전체 이벤트 리스트 + "다음 경기 찾기" CTA (홈 탭 유도)
```

#### 3-6. 설정 탭

```
설정
├── 관심 리그/팀 관리 (추가/삭제)
├── 알림 On/Off (전체)
├── 햅틱 패턴 가이드 (→ 아래 3-7로 이동)
└── (앱 정보, 버전 등 최소 정보)
```

#### 3-7. 햅틱 패턴 가이드 화면

```
햅틱 패턴 가이드
├── 이벤트별 리스트 (골 / VAR / 옐로카드 / 레드카드 / 하프타임 / 종료)
└── 각 항목 탭 → 해당 진동 즉시 재생 (반복 학습 가능)

* 온보딩에서 본 프리뷰를 언제든 다시 체험할 수 있게 하는 용도
* "패턴이 헷갈릴 때 여기서 복습" 이라는 명확한 목적
```

### 4. IA 다이어그램 (전체 요약)

```
[최초 실행]
                              │
                        온보딩 플로우 (5단계)
                              │
                              ▼
        ┌─────────────────────────────────────┐
        │              메인 탭 구조                │
        │  ┌────────┬──────────┬──────────┐   │
        │  │  홈     │  팔로잉    │   설정     │   │
        │  └────────┴──────────┴──────────┘   │
        └─────────────────────────────────────┘
                  │              │            │
                  ▼              ▼            ▼
           경기 상세 (모달)  경기 상세 (모달)  ├─ 팀/리그 관리
           (홈에서 발견)   (팔로잉에서 접근)  ├─ 알림 On/Off
                  │              │            └─ 햅틱 패턴 가이드
                  └──────┬───────┘                   │
                         ▼                            │
                 [팔로우 토글]                          │
                 [이벤트 요약 리스트]  ◄──────────────────┘
                                    (동일 화면 재사용)
```

### 5. 설계 원칙 
| 원칙 | 적용 |
| --- | --- |
| 탭 최소화 (3개) | 홈/팔로잉/설정 외 뉴스, 커뮤니티 탭 등 일체 배제 |
| Depth 최소화 | 홈 → 경기 상세까지 2단계 이내, 그 이상 깊어지는 화면 없음 |
| "확인용 화면"과 "실시간 화면"의 분리 | 앱 내 어디에도 실시간으로 계속 지켜보게 만드는 라이브 스코어보드 UI 없음 (경기 상세는 열었을 때만 스냅샷 확인) |
| 학습 보조 화면 별도 존재 (햅틱 가이드) | 패턴이 헷갈리는 실패 케이스(⑤ User Flow C)를 위한 안전장치 |
