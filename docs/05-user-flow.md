# 4. 유저 플로우

### A. 온보딩 & 경기 등록 흐름

```
앱 실행 (최초)
  ↓
온보딩 스플래시
  "화면을 보지 않아도, 경기의 흐름을 몸으로 느껴보세요"
  ↓
햅틱 프리뷰 체험
  (골 / VAR / 카드 진동을 미리 느껴보는 화면, 2~3개 샘플)
  ↓
리그 선택
  ┌─ EPL
  └─ FIFA World Cup
  ↓
관심 팀 선택
  (선택한 리그 내 팀 목록에서 1개 이상 선택)
  ↓
알림 권한 요청 (iOS Push Permission)
  ┌─ 허용 → 다음 단계
  └─ 거부 → 안내 화면 ("알림 없이는 핵심 기능 사용 불가, 설정에서 변경 가능")
  ↓
[신규] 워치 무음 모드 안내 (Apple Watch 페어링 감지 시에만 노출)
  "회의 중에도 몰래 확인하고 싶다면?"
  안내: "Apple Watch를 무음 모드로 설정하면 소리 없이 진동만 느낄 수 있어요"
  → 설정 방법 간단 안내 (텍스트 또는 짧은 이미지 가이드)
  → CTA: "확인했어요" (워치 미착용/미감지 시 이 스텝 자동 스킵)
  ↓
홈 화면 진입
  ↓
오늘/예정 경기 목록 확인
  (관심 팀이 포함된 경기가 상단 하이라이트)
  ↓
경기 선택 → 상세 화면
  ↓
"이 경기 팔로우하기" 탭
  ↓
경기 팔로우 완료
  (등록 확인 + "경기 시작 10분 전 알림 예정" 안내)
  ↓
[대기 상태] → B 흐름으로 연결
```

> **신규 스텝 설계 근거**: 페르소나 1(직장인 "몰래 확인러")의 핵심 시나리오는 회의 중 조용히 진동만 느끼는 것인데, 워치 사운드 파형이 실제 소리로 재생될 경우 이 경험이 깨질 수 있습니다. 사운드 파일 자체를 최소 음량으로 설계(⑧ Haptic Design)하는 것만으로는 100% 보장이 안 되므로, 온보딩에서 무음 모드 설정을 적극 안내하는 것을 P0 플로우에 포함합니다. 단, Apple Watch를 착용하지 않은 사용자에게는 불필요한 스텝이므로 페어링 감지 시에만 조건부로 노출합니다.

### B. 경기 진행 중 흐름 (앱을 열지 않은 상태 기준)

```
[백그라운드 대기 상태]
  ↓
경기 시작 10분 전
  → Push + 사운드 파형 기반 진동: "곧 경기가 시작됩니다" (kickoff_pre.caf 또는 별도 파일)
  ↓
경기 시작 (Kick-off)
  → Push + 진동: kickoff.caf 패턴
  ↓
──────────── 전반전 진행 중 ────────────
  ↓
  이벤트 발생 시마다 (비동기, 반복)
  ┌─────────────────────────────────┐
  │  골 발생        → goal.caf 패턴        │
  │  VAR 판독 중    → var.caf 패턴         │
  │  옐로카드       → yellow_card.caf 패턴  │
  │  레드카드       → red_card.caf 패턴     │
  └─────────────────────────────────┘
  ↓
전반 종료 (Half-time)
  → Push + 진동: halftime.caf 패턴
  ↓
──────────── 후반전 진행 중 ────────────
  ↓
  (전반전과 동일한 이벤트 루프 반복)
  ↓
경기 종료 (Full-time)
  → Push + 진동: fulltime.caf 패턴
  ↓
[선택적] 사용자가 알림 탭 or 앱 실행
  ↓
경기 결과 화면
  (최종 스코어, 주요 이벤트 타임라인 텍스트 요약)
  ↓
다음 경기 등록 유도
  (홈 화면으로 복귀, 다가오는 경기 리스트 노출)
```

### C. 진동 이벤트 발생 시 사용자의 실제 인지 흐름 (마이크로 플로우)

```
[화면을 보지 않는 상태, 워치는 무음 모드 설정 완료 상태]
  ↓
손목/주머니에서 진동 감지 (사운드 파형 기반, 소리는 최소화됨)
  ↓
패턴 인지 (학습된 리듬 매칭)
  ↓
  ┌── 패턴이 명확히 구분됨 → "아, 골이구나" 즉시 인지 (목표 상태)
  │
  └── 패턴이 헷갈림 → 화면 확인 충동 발생 (실패 상태, 최소화해야 할 케이스)
  ↓
(선택) 확인하고 싶으면 → 화면 켜서 앱 열기 → 경기 상세/타임라인 확인
(선택) 확인 안 해도 됨 → 그대로 하던 일 지속 (성공 상태)
```

---

### Flow 설계 강조점 (업데이트)

1. **A 흐름은 여전히 최대한 짧게**: 무음 모드 안내 스텝이 추가됐지만, 조건부 노출(워치 미착용 시 스킵)로 대부분 사용자에게는 흐름 길이가 그대로 유지됩니다.
2. **B 흐름은 전부 사운드 파형 기반**: Core Haptics(Foreground 전용)가 완전히 배제되었으므로, B 흐름의 모든 진동은 예외 없이 `.caf` 파일 재생 방식입니다.
3. **C 흐름의 전제 조건이 명확해짐**: "성공 상태"에 도달하려면 ① 패턴 자체가 구분되어야 하고 ② 무음 모드가 설정되어 있어야 함 — 이 두 조건이 충족되어야 페르소나 1의 시나리오가 완성됩니다.
