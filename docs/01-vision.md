# 1. 프로젝트 비전

## ① Project Vision

### Haptic Football — Screenless Football Experience

---

#### 1. 왜 만드는가 (Why)

축구를 좋아하는 사람에게 "지금 경기가 어떻게 흘러가고 있는지"는 실시간으로 느끼고 싶은 정보입니다. 하지만 현실에서는 화면을 볼 수 없는 순간이 훨씬 많습니다.

- 회의 중인 직장인
- 지하철/버스로 이동 중인 사람
- 수업 중인 학생
- 손을 쓸 수 없는 운전 중인 사람
- 눈을 감고 쉬고 싶은 사람

기존 축구 알림 앱들은 "골이 들어갔습니다"라는 텍스트 푸시를 보내지만, 결국 사용자는 **화면을 켜고, 잠금을 풀고, 앱을 열어서 확인**해야 합니다. 이 순간 이미 "경기를 몰래, 조용히, 방해받지 않고 따라가고 싶다"는 원래의 니즈는 깨집니다.

Haptic Football은 이 마지막 연결고리 — "화면을 봐야 한다"는 전제 자체를 없애는 것을 목표로 합니다. 진동 패턴만으로 골, VAR, 위험한 상황, 카드, 하프타임, 경기 종료를 몸으로 느끼게 함으로써, 사용자는 **눈을 감고도, 화면을 켜지 않고도, 소리를 내지 않고도** 경기의 흐름 속에 머물 수 있습니다.

#### 2. 어떤 문제를 해결하는가 (Problem)

| 기존 앱의 한계 | Haptic Football의 해결 방식 |
| --- | --- |
| 알림이 와도 화면을 봐야 내용을 알 수 있음 | 진동 패턴 자체가 정보 → 안 봐도 됨 |
| 모든 알림이 동일한 진동/소리 (구분 불가) | 이벤트별로 완전히 다른 햅틱 패턴 |
| 회의/수업 중엔 사실상 무용지물 | 조용하고 은밀하게 손목/주머니로 인지 가능 |
| 정보 과부하 (문자, 스코어, 통계까지 다 보여줌) | 딱 "흐름"만 전달, 나머지는 의도적으로 생략 |
| 워치 앱이 있어도 결국 화면 의존 | 워치의 물리적 특성(항상 몸에 붙어있음)을 진동 채널로 활용 |

#### 3. 누가 쓰는가 (Who)

**핵심 타겟**: 축구를 "라이브로 계속 따라가고 싶지만, 화면을 볼 수 없는 상황에 자주 놓이는" 팬.

특정 인구통계보다 **상황(Context) 기반 타겟팅**에 가깝습니다.

- 특정 팀/리그의 강성 팬이라기보다, "이 경기 결과가 실시간으로 궁금한" 라이트~미들 팬층까지 포괄
- Apple Watch를 이미 착용하고 있는 사용자 (초기 MVP 핵심 세그먼트)

(자세한 페르소나는 다음 문서 ②에서 다룹니다.)

#### 4. 기존 앱과 무엇이 다른가 (Differentiation)

| 구분 | 기존 축구 앱 (예: OneFootball, FotMob 등) | Haptic Football |
| --- | --- | --- |
| 정보 전달 채널 | 텍스트 푸시 + 화면 UI | 진동 패턴 (화면 없이도 이해 가능) |
| 정보량 | 최대한 많이 보여주려 함 (통계, 순위, 뉴스 등) | 의도적으로 최소화, 흐름 전달에 집중 |
| 사용 맥락 | "확인하러 가는" 능동적 행위 필요 | "느껴지는" 수동적/주변적 인지 |
| 핵심 디바이스 | 스마트폰 | 스마트폰 + 워치 조합 (워치가 진짜 주인공) |
| 경쟁 축 | 정보의 양과 속도 | 정보의 질과 "안 봐도 되는 경험" |

Haptic Football은 "더 많은 정보를 더 빠르게" 경쟁하는 시장에 들어가지 않습니다. 대신 "정보를 얼마나 안 보고도 느낄 수 있는가"라는, 아무도 제대로 풀지 않은 축을 선점합니다.

#### 5. 한 줄 비전 (North Star)

> **"화면을 보지 않아도, 경기가 지금 어떻게 흘러가고 있는지 몸이 먼저 안다."**
> 

#### 6. 성공의 정의 (MVP 관점)

MVP 단계에서 성공은 다운로드 수나 리텐션 지표가 아니라, 다음 한 문장으로 정의합니다.

> **한 명의 사용자라도, 화면을 보지 않고 손목의 진동만으로 골을 알아채고 "이거 진짜 편하다"고 말하는 순간을 만든다.**
> 

이 한 순간이 검증되면, 그다음에 확장(리그 추가, 정교한 통계, 커뮤니티 기능 등)을 고민합니다.
